Plant Species Database

A searchable database application for cataloguing and managing plant species information, designed to help users efficiently store, retrieve, and update plant data.

Background

This project was developed as part of CS11002 using java. The assignment brief required us to create an application that could store data about user inputted plants.

As this was somewhat of a prototype, it was not fully designed in advance. As a result of this, the code was added in pieces on top of each other, leading to repeated sections of code, and an overall slightly inefficient and messy structure.

Some features are not fully implemented to the best of my ability due to time constraints and my lack of knowledge of Java at the time, as it was my first big project in the language.

Features

Database

In the background, this program runs off of an arraylist of Plant objects. Each object within the arraylist contains information related to a specific plant species, which is entered by the user.

This mimics a real database system, with each plant object containing fields such as species name, leaf area, etc.

The use of an arraylist allows for easier addition and removal of plant entries, as well as efficient sorting and searching.

Plant object structure

Menu System

Menu system

The entire program revolves around the menu, which calls certain methods and functions when options are selected.

Due to the way it was implemented, if in future a second database was to run off of the same program, then the two menus could co-exist with no problems.

Adding and Deleting Plants

Add new plant entries to the collection or remove existing ones as needed, keeping the database up to date.

Adding and deleting plants

Sorting the Collection

Sorting the collection

The database menu contains multiple options for how the user might want to sort the database. This is then committed to the arraylist, and the resulting collection can be viewed by the user.

Viewing the Whole Collection

To display the entire collection of plants, the program uses a combination of multiple methods to create a table and display it to the user.

Viewing the collection

Calculating the Evaporation Rate

The program works in the background to calculate both the leaf area and the evaporation rate. This is done in the background whenever the user updates either the leaf length or width.

The results from these calculations are shown in the table whenever the user either searches for a plant, or views the entire collection.
